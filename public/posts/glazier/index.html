Getting started with Windows Imaging & Glazier

What is Glazier?

From https://github.com/google/glazier#glazier and https://github.com/google/glazier#why-glazier:

Glazier is a tool developed at Google for automating Windows operating system deployments.

Glazier was created with the following 3 core principles in mind: Text-Based & Code-Driven, Scalability, and Extensibility.

Sounds awesome. How do I get started?

The Glazier readme suggests:

See our docs site for how you can get started with Glazier.

The setup overview, however, leaves a lot of the setup details (WinPE, portable Python, drivers, etc…) as an exercise for the reader. Hence why I am writing this blog post: I hope to demystify some of the setup details and share a reproducible way to create the WinPE environment needed to run Glazier.

Ironically, I may leave some setup details as an exercise for you (ex. creating a Windows 10 virtual machine). But please feel free to reach out to me or submit a pull request if you need more details!

Setting up Glazier

In the future, I may create a Terraform configuration & a Powershell script to automate most of this but, for fear of analysis paralysis, I’ll be doing this by hand for now. This should still be quite reproducible.

Installing the build environment dependencies

Note: Chocolatey is not strictly required to get Windows ADK installed but installing Windows ADK and the WinPE add-on manually is daunting. Chocolatey makes it very easy.

  1. Set up a Windows 10 virtual machine. There many ways to do this: I’ll be using VMWare Fusion on my Macbook.

  2. Launch Powershell as an Administrator and install the awesome OSD Powershell module. Mad props to @SeguraOSD for this wizardry.

    Install-Module OSD -Force

    If prompted, enter Y to install other depedencies.

  3. As a security best practice, inspect https://community.chocolatey.org/install.ps1 and make sure it’s not doing anything fishy. If you are satisfied, save a private copy of the install.ps1 script somewhere and run it. Or if you are lazy, just curl it from the internet. You should feel bad though :P

    Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))

  4. Relaunch Powershell (again as an Administrator) so that the choco command becomes available.

  5. Run choco install windows-adk -y to install the Windows ADK.

    ...
The install of windows-adk was successful.
Software installed as 'exe', install location is likely default.
Chocolatey installed 1/1 packages.
See the log for details (C:\ProgramData\chocolatey\logs\chocolatey.log).

  6. Run choco install windows-adk-winpe -y to install the Windows ADK WinPE add-on.

    ...
The install of windows-adk-winpe was successful.
 Software installed as 'exe', install location is likely default.

Chocolatey installed 1/1 packages.
See the log for details (C:\ProgramData\chocolatey\logs\chocolatey.log).

Setting up WinPE development tools

Note: OSDCloud is, within itself, a very cool and powerful imaging tool that you can use without Glazier. And OSDCloud might be a better choice than Glazier for many admins. However, today, I’m going to borrow it’s sorcery to help us bootstrap Glazier :) Additionally, OSDCloud probably adds lots of stuff to our image that we don’t actually need for Glazier. But for simplicity sake, we’ll leave in our WinPE.

  1. Run powershell.exe -ExecutionPolicy Bypass to launch a new Powershell session which will allow us to run scripts. I’m sure there is a better way, but /shrug. If you know a more proper way to do this, let me know!

  2. Import the OSD Module: Import-Module OSD

  3. Run New-OSDCloud.template -WinRE -Verbose to generate a “Universal WinPE”. This may take a while so grab a coffee :) The -WinRE flag adds all the components necessary to utilize Wifi in WinPE! Neato! Credits to Oliver Kieselbach. To learn more, check out https://osdcloud.osdeploy.com/get-started/new-osdcloud.template/winre.

    Important Note: This command pulls some resources from https://github.com/OSDeploy/OSDCloud/ and other places on the internet. Audit the repositories as needed.

    Once this command is finished, you should see:

    2021-05-02-180823 New-OSDCloud.template Completed in 05 minutes 25 seconds
OSDCloud Template created at C:\ProgramData\OSDCloud
Get-OSDCloud.template will return C:\ProgramData\OSDCloud
